PROJECT 01

RecipeBox

Advanced Web Accessibility (WCAG 2.1 AA Compliance)

View Live Demo

Project Overview

RecipeBox is a fully responsive, five-page web application engineered to demonstrate advanced web accessibility implementation and document remediation techniques. The platform features a recipe search dashboard, detailed recipe views, an interactive weekly meal planner, a dynamic shopping list, and an ingredient submission form.

The primary objective was to build a visually engaging, modern "dark mode" interface without compromising strict WCAG 2.1 AA accessibility standards—proving that premium aesthetics and rigorous accessibility are not mutually exclusive.

Key Accessibility Achievements

The application was architected to ensure it is fully perceivable, operable, understandable, and robust for users relying on assistive technologies including screen readers, keyboard navigation, and voice control systems.

Semantic Architecture

Strict HTML5 landmarks (<header>, <main>, <nav>, <footer>) with logical heading hierarchy (H1-H4) for seamless screen reader navigation.

Keyboard Navigation

Visually hidden "Skip to main content" link, logical tab order, and custom high-contrast focus indicators for keyboard-only users.

Accessible Data Tables

Complex visual data converted to semantic HTML tables with proper <caption>, <thead>, and scope attributes.

Dual Data Representation

CSS-based nutritional charts paired with .sr-only text providing exact values to screen reader users.

Interactive Accessibility (ARIA)

Dynamic State Announcements

Strategically utilized aria-live="polite" regions to announce dynamic UI changes, such as real-time shopping list updates ("1 of 4 items checked"), ensuring non-visual users receive immediate feedback.

Advanced Form Handling

Built a sophisticated multi-section "Add Recipe" form with dynamic ingredient row addition. All inputs are explicitly linked to visible <label> elements, with required fields marked using aria-required, and inline validation errors connected via aria-describedby and role="alert".

Accessible Custom Controls

Engineered custom interactive elements including expandable filter panels (<details> / <summary>) and dietary toggle buttons using native, semantic HTML controls—guaranteeing built-in screen reader compatibility and keyboard support.

Inclusive Visual Design

Color Contrast & Typography

Developed a premium dark-mode aesthetic built on CSS Custom Properties. Rigorously verified that all text, interactive elements, and focus indicators exceed WCAG AA contrast requirements (4.5:1 for normal text, 3:1 for large text and UI components).

Reduced Motion Support

Implemented prefers-reduced-motion media queries to automatically detect user preferences and disable animations for users with vestibular disorders or motion sensitivity.

Value Delivered

RecipeBox serves as a production-ready demonstration of how inclusive design principles can be seamlessly integrated into complex web applications without sacrificing visual appeal. By prioritizing accessibility from the initial design phase, the project proves that accessible design enhances the experience for all users through clearer information architecture, more robust interaction patterns, and superior semantic structure.

PROJECT 02

ClinOps

Agentic Clinical Trial Project Manager

Watch Demo Video

Overview

Clinical trials frequently suffer from fragmented tools, resulting in operational delays, missed compliance steps, and elevated inspection risks. ClinOps addresses these challenges by delivering a unified, AI-powered workspace that functions as an intelligent copilot for clinical operations.

The platform reads complex clinical trial protocols, translates them into actionable milestones, and proactively identifies compliance risks. ClinOps empowers trial coordinators and regulatory advisors by streamlining audit preparation and document control—all while preserving their specialized medical and legal judgment.

Web Design & User Experience

The web application was meticulously designed to handle complex clinical data intuitively, prioritizing performance, clarity, and responsiveness across all user workflows.

Modern, High-Performance Stack

Built on Next.js 15 (App Router) with TypeScript, leveraging Server-Side Rendering (SSR) for lightning-fast initial page loads and optimal SEO. Advanced code splitting and strict type safety ensure seamless scalability even with thousands of documents and protocol dependencies.

Tailored, Role-Based Interfaces

The UI dynamically adapts based on the authenticated user's persona to reduce cognitive overload:

Trial Coordinators

Access operation-centric tabs: Trial Overview, Task Checklists, Trial Timeline, Patient Enrollment Tracking

Regulatory Advisors

Access compliance-centric tabs: Protocol Requirements, Risk & Controls, Audit Preparation, Documentation Review

Sophisticated Styling & Responsive Design

Utilizing Tailwind CSS, the system features a clean, accessible design that adapts beautifully across desktop, tablet, and mobile viewports with a professional, data-forward aesthetic specifically suited for enterprise healthcare software.

Interactive Dashboards & Smart Widgets

The core dashboard elegantly visualizes enrollment status, milestone progress, and compliance tasks. A standout innovation is "Smart Send"—allowing users to automatically generate and pin structured dashboard widgets directly from AI chat insights, transforming conversational intelligence into persistent, actionable visualizations.

Seamless AI Copilot Integration

The AI is deeply integrated as a context-aware clinical assistant. The UI maintains a natural split-view layout, enabling users to reference trial documents while simultaneously conversing with the AI to generate content, assess risks, or extract protocol requirements.

Optimized Interactive Components

Complex UI components leverage advanced React patterns including useMemo and useCallback to prevent unnecessary re-renders, ensuring exceptional performance even when rendering hundreds of document version histories and protocol dependencies.

Reliability & Observability

To guarantee stability in mission-critical clinical settings, the UI is supported by 167 passing tests spanning Unit, Integration, and End-to-End UI suites via Jest and React Testing Library. The application is fully instrumented with OpenTelemetry, feeding real-time metrics into Prometheus, Grafana, and Loki for comprehensive monitoring.

Measurable Impact

ClinOps fundamentally transforms clinical trial operations by reducing the "Protocol-to-Plan" conversion timeline through intelligent automation. Early adopters report substantial reductions in administrative overhead and significantly improved audit readiness, with compliance documentation preparation time cut by more than half.
